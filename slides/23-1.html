Interprocess Communication Devices

  • Pipes and Named Pipes (FIFOs)
    • Pipes: Allow one-way communication between processes. They are commonly used for communication between a parent process and its child process.
    • Named Pipes (FIFOs): Similar to pipes but with a name in the filesystem, allowing unrelated processes to communicate.
  • Shared Memory: Allows multiple processes to access the same memory space, enabling high-speed data exchange. Synchronization mechanisms like semaphores or mutexes are often required to manage access to shared memory.
  • Message Queues: Provides a way for processes to exchange messages in a controlled, asynchronous manner. Messages are stored in a queue and can be read in a first-in, first-out (FIFO) order.
  • Semaphores: Used for controlling access to shared resources by multiple processes. Semaphores are typically used to signal the availability or unavailability of resources.
  • Signals: Used to notify a process that a particular event has occurred. Signals can be used for interprocess communication, but they are more commonly used for handling asynchronous events.
  • Files: Processes can communicate by reading from and writing to shared files. This method is simple but not as efficient as other IPC mechanisms.
  • Memory-Mapped Files: Allows files or devices to be mapped into memory, enabling processes to perform I/O operations directly on memory, which can be faster than using traditional read/write operations.
  • Remote Procedure Calls (RPC): Allows a program to cause a procedure to execute in another address space (commonly on another physical machine) as if it were a local procedure call.
  • D-Bus: A message bus system that provides a simple way for interprocess communication. It is widely used in Linux environments for communication between system services and applications.
  • CORBA (Common Object Request Broker Architecture): A standard defined by the Object Management Group (OMG) designed to facilitate communication between various applications, regardless of the languages they are written in or the platforms they are running on.
  • Thrift: A software framework for scalable cross-language services development, developed by Facebook. It combines a software stack with a code generation engine to build services that work efficiently and seamlessly between languages.
  • HTTP/REST APIs: Processes can communicate over HTTP using RESTful web services. This method is commonly used for communication between distributed systems and microservices.
